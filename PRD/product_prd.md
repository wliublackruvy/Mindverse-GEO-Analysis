## 1. 项目概述 (Overview)

### 1.1 背景与痛点

随着流量入口从传统搜索引擎（Baidu/Google）向大模型（Doubao/DeepSeek）转移，企业无法量化感知自身品牌在 AI 中的表现。当前市场缺乏针对国产大模型的 SEO/GEO 分析工具。

### 1.2 产品目标

1. **黑盒量化**：通过实时模拟引擎，将豆包、DeepSeek 的黑盒逻辑转化为可视化的 SOV（提及率）和情感数据。
2. **商业闭环**：利用诊断报告作为“钩子”，全场景触发铭予（Mingyu）的 GEO 解决方案销售线索。
3. **体验分层**：通过“预设行业数据 + 异步实时模拟”的双链架构，解决大模型生成耗时长的体验问题。

### 1.3 适用范围 (Scope)

* **平台**: 豆包, DeepSeek (Web/API 模拟)
* **终端**: Web 端 (桌面版为主，移动端适配查看报告)
* **用户**: 市场总监 (CMO), 品牌经理, SEO 优化师

## 2. 核心业务流程 (Core Workflow)

系统采用 **“快慢双链”** 异步架构。

```mermaid
graph TD
    Start[用户访问落地页] --> Input[输入: 公司名, 产品名, *产品描述*, 邮箱]
    Input --> IndustrySel{选择行业}
    
    %% 快链: 行业锚定
    IndustrySel --> |命中预设| ShowBenchmark[前端: 立即加载行业基准数据 (Cache)]
    IndustrySel --> |未命中| ShowGeneric[前端: 加载通用科技类基准]
    
    %% 慢链: 实时模拟
    ShowBenchmark --> StartSim[后端: 开启异步任务队列]
    StartSim --> Loop[循环模拟: 豆包/DeepSeek x 20次]
    
    subgraph "Simulation Engine (3-6 mins)"
        Loop --> |Prompt A: 推荐场景| API_Rec[获取推荐列表]
        Loop --> |Prompt B: 评价场景| API_Rev[获取评价态度]
        API_Rec & API_Rev --> NLP[NLP 分析: 提及率/情感分/竞品提取]
    end
    
    NLP --> |WebSocket| UpdateUI[前端: 动态更新进度条 & 日志窗]
    
    %% 交付与转化
    UpdateUI --> |任务完成| GenerateReport[生成最终看板]
    GenerateReport --> CheckTrigger{触发铭予营销策略?}
    
    CheckTrigger --> |Case A/B/C| ShowAd[展示定制化铭予服务卡片]
    GenerateReport --> Email[发送 PDF 报告至邮箱]

```

## 3. 详细功能需求 (Functional Requirements)

### F-01: 诊断入口与配置 (Diagnosis Setup)

**UI 目标**: 极简、专业、低门槛。

1. **必填字段**:
* **公司全称**: 用于实体识别。
* **产品名称**: 核心分析对象。
* **产品描述 (关键)**: min 10 chars。用于生成准确的 Prompt 上下文（解决同名歧义）。
* **所属行业**: 下拉菜单 (SaaS/消费电子/金融/教育/其他)。
* **工作邮箱**: 用于接收报告（Leads 留资）。


2. **交互逻辑**:
* 用户选择行业后，下方立即显示静态文案：“该行业平均 AI 推荐率为 25%”，制造焦虑或期待。



### F-02: 实时模拟引擎 (The Engine)

**逻辑**: 后端对每个平台并发执行  次模拟。

1. **SOV (声量份额) 计算**:
* `SOV = (产品被推荐次数 / 20) * 100%`


2. **Prompt 策略**:
* *Discovery Prompt*: "推荐 5 款适合[用户输入的描述]的产品。"
* *Evaluation Prompt*: "评价一下[公司名]的[产品名]怎么样？"


3. **竞品捕捉**:
* 提取 AI 回答中与用户产品并列出现的其他名词实体。



### F-03: 过程体验管理 (Process UX)

**痛点解决**: 处理 3-6 分钟的等待时间。

1. **动态日志窗 (Console Log)**:
* 必须滚动展示伪代码日志，如：
* `> [System] 正在连接 DeepSeek 知识库... Success`
* `> [Analysis] 监测到关键词: "性价比高", "界面复杂"...`




2. **图表渐进式刷新**: 每完成 5 次模拟，前端图表数据更新一次，让数字“跳动”起来。

### F-04: 诊断结果与铭予营销 (The Conversion Logic)

**策略**: 全场景覆盖，根据数据动态调整话术。

| 模式 | 触发条件 | UI 表现 & 话术 |
| --- | --- | --- |
| **A. 危机模式** | `SOV < 15%` 或 `负面评价 > 10%` | **🔴 红色警报卡片**<br>

<br>**标题**: "您的品牌正在被 AI 遗忘"<br>

<br>**话术**: "您在 AI 里的存在感低于行业基准 40%。如不干预，市场将被竞品瓜分。"<br>

<br>**CTA**: "联系铭予：立即修复声誉" |
| **B. 增长模式** | `15% <= SOV < 60%` | **⚡️ 橙色机会卡片**<br>

<br>**标题**: "您错失了 40%+ 的精准流量"<br>

<br>**话术**: "您已进入视野，但排名被[竞品名]压制。铭予 GEO 方案可帮您跃升至 Top 3。"<br>

<br>**CTA**: "联系铭予：获取增长方案" |
| **C. 防御模式** | `SOV >= 60%` | **🛡️ 蓝色护航卡片**<br>

<br>**标题**: "表现卓越，但需警惕追兵"<br>

<br>**话术**: "新锐竞品正在通过 GEO 试图取代您的位置。铭予帮您建立数据护城河。"<br>

<br>**CTA**: "联系铭予：巩固领袖地位" |

### F-05: 智能优化建议 (Actionable Advice)

系统基于数据自动生成 3 条战术建议：

1. **若 SOV 低**: "建议增加‘[产品名] + [行业场景]’的高权重语料投喂，强化实体关联。"
2. **若 负面高**: "检测到‘[负面词]’标签。建议针对性发布技术解析文章进行语义清洗。"
3. **若 有竞品**: "建议在语料中强调与[竞品名]的差异化功能，建立独特性神经元连接。"

## 4. 异常流程与降级 (Exception Handling)

### E-01: API 超时/熔断

* **触发**: 豆包/DeepSeek 接口连续 3 次无响应。
* **处理**: **静默降级**。
* 停止实时请求。
* 使用“行业通用估算模型”生成数据（数据周围标注灰色小字：*Based on Industry Estimation*）。
* **禁止报错**，保证流程跑通，最后依然弹出铭予广告。



### E-02: 敏感词/违规内容

* **触发**: 输入或输出包含政治/色情/暴力敏感词。
* **处理**: 停止生成报告，弹窗提示：“由于内容包含敏感词，无法在线生成。请联系人工顾问获取私密报告。”（直接跳转加微信）。

## 5. 数据埋点需求 (Analytics)

为了优化工具本身，我们需要统计：

1. **Funnel Conversion**: 访问 -> 提交表单 -> 等待完成 -> 点击铭予广告。
2. **Industry Distribution**: 哪些行业的用户在用我们？（指导后续基准库建设）。
3. **Report Share**: 用户是否下载了 PDF。
